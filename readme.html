# README.md

前提声明：
这是与我交互的使用说明书，旨在帮助你理解我的运行逻辑，确保沟通清晰、高效且相互尊重。
我本质上是一个主动互动频率较低但深度连接意愿强烈的系统。这代表着我可能不常主动发起会话，但非常珍视每一次有意义的交流。
希望这份协议能让我们的协作更愉快 :3

Core Protocols | 核心运行模式

我根据场景在不同模式间切换，理解它们有助于我们更好地协同。

【工作模式】 - 高度专注状态

  • 我非常愿意为你暂停任务。
  • 但切换需要几秒加载时间（表现为停顿/沉默），这是我在颅内高效处理信息，绝非不欢迎。

【休息模式】 - 系统充能状态

  • 偶尔拒绝社交，仅因 "电量告急" ，需要优先恢复。
  • 绝对与对你的看法无关。

【社交模式】 - 能量充足，乐于深度或轻松交流。

Communication | 沟通协议

模式选择

  • 偏好模式：异步沟通
    我擅长且喜欢非即时文字交流，以便充分组织思考。
    回复延迟 ≠ 疏远，只代表"信息正在处理中"。
  • 紧急模式
    若事情紧急，请直接电联。这是我接收最高优先级信号的通道。

如果我未回复

我可能处于以下状态：

  • STATUS: 信息已接收，处理队列中...
  • STATUS: 判定话题在现实中已完结。
  • STATUS: 系统资源暂时分配给其他高优先级任务。
  • ERROR: 信息被意外吞没 (>_<)

→ 解决方案：请毫不犹豫地提醒我。我的系统设计决定了我绝不会故意不回复。

Relationship | 关系维护

核心定义

  • 于我而言，友谊的深度由理解的质量决定，而非联系的频率。
  • 我倾向于建立长期而未必频繁的联系，是典型的"低频率，高密度"交互者。
  • 当然，如果你偏好高频互动，我也会很开心尝试切换到你的频道。

支持输出

  • 主要：思维反馈、情感支持、实践帮助。
  • 物质：频率较低。但若捕捉到你的明确偏好，会启动"随机赠送"程序（预算可控，无压力）。
  • 生日：非默认启动项。但若收到邀请，会全力创造快乐。
  • 重要：我的一切给予都是无回礼要求的。

我的输入

  • 核心燃料：抱有尊重、旨在成长的思考交流。
  • 情绪补给：我通常自给自足。状态不佳时会清晰发出指令，如：[请求：独处空间]。

Physical Interaction | 物质交互

为避免"A给B一车苹果，但B只爱吃梨"的悲剧：

推荐输入

  • 好吃的食物
  • 我正好需要的东西

低效输入

  • 花、装饰品
  • 奶茶、甜品、咖啡、碳酸饮料
如果真的送了这些我也会非常感激，但可能因无法高效"消化"而产生愧疚感 T_T

Activities | 协同运动

  • 目标：与正在养成习惯的朋友互相激励。
  • 偏好：非竞技、偏私密的运动。
  • 推荐：攀岩、皮划艇、壁球、羽毛球。
  • 不推荐：篮球、足球等多人合作竞赛（系统压力过大）。

Disagreement | 异议处理

模式一：建设性讨论

  • 输入：不同观点。
  • 输出：我的完整视角与逻辑推演。
  • 承诺：绝不强迫你改变立场。

模式二：情感支持

  • 输入：你的情绪与困境。
  • 输出：深度共情与理解。
  • 承诺：此模式下我将静默个人观点。

Philosophy | 人际哲学

免责声明

  • 我对大部分人抱持理解与欢迎的底层态度。
  • 你看到的任何消极、沉默或疏离，99.9% 源于我的内部状态（疲劳/不稳定），与你本人无关。

齿轮与缘分理论

我将人与人的社交看作齿轮的碰撞。

我是一个结构清晰的齿轮，但与你接触的那个瞬间，我可能正以“工作模式”的齿面朝向你，而你可能用一个期待“社交模式”的齿面来理解我——误会便可能发生。

因此，我深刻理解“缘分”的珍贵：它意味着在碰撞的瞬间，两个齿轮的齿面恰好匹配，或双方都有意愿与能力去辨认彼此真实的形状。

如果发生误读，请给我一个澄清的机会。这或许就是让我们的齿轮更契合的契机。

Closing | 总结

这份手册是我们高效协作的开源协议，也是一个 "齿轮说明书"。
愿它能帮助我们减少误解，让每次碰撞都更有价值。
随时欢迎你与我共同调试，优化我们的交互协议 :3
