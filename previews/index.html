S1 · Manual · 协议 →

The Manual

齿轮说明书

把整个首页当成一份被认真记录的自我。README.md 的元素延展到全站气质。

  • 左侧 4.5rem 是装订线，唯一的竖线
  • §00–§07 章号挂在边距里
  • 没有卡片框：说明书的章节不放盒子里
S2 · Density · 高密度 →

Low frequency, high density

沉默不是缺席，是听

段间巨大留白，每段内部很重。线少了，才贵——只有一根细线分段。强迫慢读。

  • 单列窄测度（36rem），强制行宽节奏
  • 6rem 段间距，是社交节奏的视觉化
  • 无侧栏，全部纵向流入
S3 · Crack · 缝隙 →

Find a crack

找一道缝隙，做一点小事

整页一根竖线偏左 1/3，内容只在右 2/3 那条"缝"里。空白是世界的大，内容是缝。

  • 不对称：左列稀疏方向标签，右列承重
  • 那根竖线就是"缝隙"本身
  • 左侧固定 sticky，作为阅读时的方向感
S4 · Log · 日志 →

A captain's log

输入、输出间的迭代

每一行都被日期锚定。Sections、Updates、Inbox 共享同一种行格式——它们都是同一本日志里的条目。

  • 最左 6.5rem 固定日期列
  • 章节卡用"last touched"日期作行键
  • 整页读起来像 git log
S5 · Frontispiece · 扉页 →

For a future stranger

给未来某个陌生人的启发

居中、窄列（32rem）、像翻开一本书的扉页。用排版的礼仪让初到者先慢下来。

  • imprint · 标题 · 短分隔线 · abstract · 目录
  • 测度窄到强迫一行不超过 28 个汉字
  • Updates / Inbox 退到下方双列页脚
S6 · Grid · 模数 →

Discipline as care

在不规则世界里建一个圆

12 栏不可见纪律——hero 占 8 栏，intro 占 4 栏靠右，cards 6+6，下方 7+5。结构本身就是声明。

  • 没有可见网格线，但每个元素都掐在栏位上
  • 呼应约束手套项目里"结构性的关怀"
  • 最适合长期扩展——新内容直接落进栏位
